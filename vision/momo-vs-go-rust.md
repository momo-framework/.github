# 2. Momo vs Go/Rust

## Сравнение Time-to-Market (TTM)
В языках Go и Rust много времени уходит на "борьбу" с языком (управление памятью, отсутствие привычных ORM).

| Язык | Скорость написания фичи | Сложность поддержки |
| :--- | :--- | :--- |
| **Go** | Средняя | Сложно (много шаблонного кода) |
| **Rust** | Низкая | Очень сложно (строгий компилятор) |
| **Momo (PHP)** | **Очень высокая** | **Легко (модульный монолит)** |

## Наш путь
Мы берем лучшее: **гибкость PHP для бизнеса** и **скорость Go для ядра**. Если функционал не требует 100k RPS, он остается на PHP.
